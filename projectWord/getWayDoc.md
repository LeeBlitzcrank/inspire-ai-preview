# 网关完整功能拆解（你的项目网关 = Spring Cloud Gateway）
网关是**所有前端请求的唯一入口**，用户H5、管理员后台全部先访问网关，网关做完校验再转发给后端各个微服务（用户服务、AI服务、灵感服务、管理服务）。
四大核心功能：跨域处理、JWT登录鉴权、接口限流、路由分发，逐个讲清楚做什么、为什么做、业务场景。

## 一、路由分发（最基础功能）
### 作用：请求分流，把不同接口转给对应服务
后端拆分成多个独立服务：
1. 用户服务：注册、登录、个人信息
2. AI灵感服务：Codex生成灵感
3. 灵感核心服务：发布、收藏、点赞、详情、ES搜索
4. 推荐推送服务：首页个性化推荐、消息推送
5. 管理员服务：监控、手动推送、配置管理

前端只有一个域名，网关根据接口路径转发：
- `/api/user/**` → 转发到 用户服务
- `/api/ai/**` → 转发到 AI灵感服务
- `/api/inspire/**` → 转发到 灵感服务
- `/api/recommend/**` → 转发到 推荐服务
- `/api/admin/**` → 转发到 管理员后台服务

### 开发要做的事
1. 配置路由规则（yml/代码两种方式）；
2. 服务发现（Nacos/直接写固定地址，本地开发用固定地址）；
3. 负载均衡：多实例服务轮流转发请求。

### 举例
前端请求 `http://127.0.0.1:8080/api/ai/generate`
网关匹配 `/api/ai/**`，自动转发到AI服务端口，前端不用关心各个服务真实地址。

## 二、跨域处理（解决前端浏览器报错）
### 背景
前端页面域名 `localhost:5173`，网关地址 `localhost:8080`，浏览器同源策略会拦截请求，报跨域错误。
### 网关要做：统一添加跨域响应头
放行配置：
1. 允许所有前端域名访问（生产可限定域名）；
2. 允许携带Cookie、Token；
3. 允许GET/POST/PUT/DELETE全部请求方法；
4. 放行自定义请求头（存放JWT令牌）。

### 开发实现
写全局CORS跨域过滤器，所有接口统一处理，**不用每个微服务单独配置跨域**，统一收口。

## 三、JWT鉴权（登录权限拦截，核心安全功能）
### 业务规则（对应你的PRD）
1. 白名单接口：不需要登录
    - 注册、登录、首页公开灵感列表、ES搜索
2. 其余接口**必须携带有效JWT令牌**：AI生成灵感、发布、收藏、点赞、个人中心、推送、管理员接口
3. 校验逻辑全在网关拦截，没登录直接拦截，不会走到后端业务服务，节约服务性能

### 网关完整校验流程
1. 从请求Header取出token；
2. 判断是否存在：不存在直接返回 401 未登录；
3. 校验token是否过期、签名是否合法；
4. 解析token拿到 userId，放入请求头向下游微服务传递；
5. 黑名单校验：已退出登录、封禁用户的token直接拦截。

### 开发要做
1. 全局过滤器，在路由转发前执行鉴权；
2. 配置接口白名单（无需登录放行）；
3. JWT工具类：解析、校验、读取用户ID；
4. 对接Redis：存放过期/注销的token黑名单。

## 四、限流防刷（保护服务、控制AI调用成本，架构评审高风险点）
### 解决两个核心问题
1. 恶意高频刷接口，压垮MySQL、Redis；
2. 频繁调用Codex AI产生高额费用，限制单用户每日调用次数。

### 两种限流规则都在网关实现
1. **全局限流**：整个系统每秒最多处理多少请求，防止流量打满；
2. **用户级限流（重点）**
    - 普通接口：单用户每秒最多5次请求；
    - AI生成灵感接口 `/api/ai/generate`：单用户每日上限50次，超出直接返回提示；

### 限流实现方案
基于Redis实现分布式限流（单机网关可用本地限流，生产分布式必须Redis）
逻辑：
以用户ID为key，记录调用次数+过期时间，达到阈值网关直接拦截，不转发下游服务。

### 开发内容
1. 限流过滤器，鉴权完成后执行；
2. Redis计数器工具；
3. 可配置限流阈值（管理员后台动态调整）；
4. 限流后统一返回提示文案。

# 完整请求执行顺序（网关内部流程）
前端请求进来 →
1. 跨域处理
2. 限流校验（超出限制直接返回）
3. JWT鉴权（未登录直接返回）
4. 路由分发，转发到对应微服务
   微服务处理完成，响应原路经过网关返回前端。

# 为什么全部放在网关做，不写在各个业务服务？
1. **统一收口**：跨域、鉴权、限流只写一遍，5个服务不用重复开发；
2. **提前拦截**：非法请求、高频刷接口在网关直接阻断，不消耗业务服务、数据库资源；
3. **安全集中管控**：所有权限校验统一维护，方便修改规则；
4. 业务服务只专注业务逻辑（AI生成、发布、推荐），不用关心安全、流量控制。

# 结合你项目的开发产出物
1. Gateway启动工程；
2. 路由配置文件；
3. 跨域全局过滤器；
4. JWT鉴权全局过滤器 + JWT工具类；
5. Redis分布式限流过滤器；
6. 统一全局异常返回（401未登录、429访问过于频繁）。
