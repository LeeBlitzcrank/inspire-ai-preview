# 灵思集 — 消息通知系统

> 版本: v1.1 | 更新日期: 2026-07-02

---

## 一、功能概述

用户在平台上产生以下互动行为时，自动触发消息通知：

| 行为 | 通知类型 | 通知对象 | 通知内容 |
|------|----------|----------|----------|
| 点赞灵感 | `like` | 灵感作者 | `{用户} 点赞了你的灵感` |
| 收藏灵感 | `collect` | 灵感作者 | `{用户} 收藏了你的灵感` |
| 评论灵感 | `comment` | 灵感作者 | `{用户} 评论了你的灵感: {评论内容}` |
| 回复评论 | `reply` | 被回复者 | `{用户} 回复了你的评论: {回复内容}` |
| 关注用户 | `follow` | 被关注者 | `{用户} 关注了你` |

**用户自助操作**：已读 / 全部已读、点击通知跳转对应灵感详情。

## 二、数据库

### 表：`user_notification`

```sql
CREATE TABLE `user_notification` (
  `id` BIGINT NOT NULL COMMENT '雪花通知ID',
  `user_id` BIGINT NOT NULL COMMENT '通知接收者',
  `type` VARCHAR(16) NOT NULL COMMENT '通知类型: like/collect/comment/reply/follow',
  `actor_id` BIGINT NOT NULL COMMENT '触发者用户ID',
  `actor_name` VARCHAR(50) DEFAULT '' COMMENT '触发者昵称',
  `content` VARCHAR(255) DEFAULT '' COMMENT '通知摘要文本',
  `target_id` BIGINT DEFAULT NULL COMMENT '关联灵感/评论ID',
  `target_title` VARCHAR(120) DEFAULT '' COMMENT '关联灵感标题',
  `is_read` TINYINT DEFAULT 0 COMMENT '0未读 1已读',
  `deleted` TINYINT DEFAULT 0 COMMENT '逻辑删除',
  `create_time` DATETIME DEFAULT CURRENT_TIMESTAMP,
  PRIMARY KEY (`id`),
  KEY `idx_user_id` (`user_id`,`is_read`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户通知表';
```

- `target_id` = 灵感 ID，前端点击通知可直接跳转灵感详情
- 自己触发的不产生通知（`actor_id == user_id` 跳过）

## 三、后端 API

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/notification/list?page=1&size=20` | 我的通知列表（分页） |
| GET | `/notification/unread` | 未读通知数量 |
| PUT | `/notification/read?notificationId=` | 标记已读（不传 id 标记全部已读） |

### 网关路由

`application.yml` 中新增路由 `inspire-notification`，将 `/api/notification/**` 转发到 inspire-core（端口 8083），同样 `StripPrefix=1` 剥离 `/api` 前缀。

```yaml
- id: inspire-notification
  uri: ${routes.core-service:http://localhost:8083}
  predicates:
    - Path=/api/notification/**
  filters:
    - StripPrefix=1
```

通知接口不在 JWT 白名单内，需要登录认证。

### 触发时机（代码位置）

1. **`InspireServiceImpl.like()`** — 点赞成功 → `notificationService.notify()`
2. **`InspireServiceImpl.collect()`** — 收藏成功 → `notificationService.notify()`
3. **`CommentServiceImpl.create()`** — 评论/回复成功 → `notificationService.notify()`
4. **`FollowServiceImpl.follow()`** — 关注成功 → `notificationService.notify()`

## 四、前端

### 4.1 首页铃铛图标

位于顶部导航 `🔍` 与 `👤` 之间，用红色数字徽标实时显示未读通知数：

- `getUnreadCount()` 在 `onMounted` 和登录态刷新时调用
- 点击跳转到 `/notifications`

### 4.2 通知页面 `/notifications`

- **顶部**：返回按钮 + 标题「消息通知」+「全部已读」按钮
- **通知列表**：按时间倒序，每条显示：
  - 类型图标（⭐ 赞 / 🔖 收藏 / 💬 评论 / ↩️ 回复 / 👤 关注）
  - 触发者用户名 + 动作摘要
  - 关联灵感标题（蓝色可点击）
  - 相对时间（`formatTime`：3分钟内→刚刚 / N小时前 / YYYY-MM-DD）
- **未读状态**：左侧蓝色小圆点 + 浅蓝背景
- **点击**：标记已读 → 跳转灵感详情
- **空态**：🔔 暂无消息通知

## 五、部署说明

1. 先执行 `user_notification` 建表 SQL
2. 重启 `inspire-core` 服务（已包含通知逻辑）
3. 重启 `inspire-gateway`（新路由生效）
4. 前端热更新或刷新页面

## 六、测试流程

1. 用户 A 登录，用户 B 登录
2. B 给 A 的灵感点赞 → A 首页铃铛显示「1」
3. A 点击铃铛 → 通知页面显示「B 点赞了你的灵感」
4. B 评论 A 的灵感 → A 收到评论通知
5. B 回复 A 的评论 → A 收到回复通知
6. B 关注 A → A 收到关注通知
7. A 点击「全部已读」→ 所有未读数清零
8. 验证不产生自我通知（自己点赞自己不通知）
