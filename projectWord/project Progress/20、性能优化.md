## 一、已完成

### 1. N+1 查询消除

**问题**：`toVO()` 对列表每项单独查询昵称（20 项 = 21 次 SQL）

**方案**：新增 `toVO(m, loginUserId, content, nicknames)` 重载，接收预查询的昵称 Map；列表方法批量查询后传入，`toVO` 直接从 Map 取值

```java
// 1 次批量查询代替 N 次逐条查询
Map<Long, String> nicknames = jdbcTemplate.query(
    "SELECT id, nickname FROM user WHERE id IN (?,?,?)", ...);
return mains.stream().map(m -> toVO(m, loginUserId, null, nicknames)).collect(...);
```

| 方法 | 优化前 | 优化后 |
|------|--------|--------|
| `listPublic()` | N 次 SQL | 2 次 SQL |
| `recommend()` | N 次 SQL | 2 次 SQL |

### 2. 缓存读接口（Spring Cache）

**方案**：在 `listPublic()` 和 `recommend()` 上添加 `@Cacheable` 注解，使用 Spring 内置 `ConcurrentHashMap` 缓存，无需 Redis 依赖

```java
@Cacheable(value = "publicList", key = "#query.page + ':' + #query.tag + ':' + #query.sort",
           unless = "#loginUserId != null")
public List<InspireVO> listPublic(InspirePageQuery query, Long loginUserId) { ... }
```

- 登录用户不缓存（因含点赞/收藏状态）
- 匿名用户缓存 60 秒

| 文件 | 改动 |
|------|------|
| `InspireCoreApplication.java` | 添加 `@EnableCaching` |
| `InspireServiceImpl.java` | `listPublic()` + `recommend()` 添加 `@Cacheable` |

### 3. ES 批量同步

**问题**：每创建/更新一条灵感就逐条索引到 ES，发布速度受 ES 响应影响

**方案**：Bulk API 批量提交，每 5 分钟全量同步一次

```java
@Scheduled(fixedDelay = 300000)
public void batchSync() {
    // 查询所有灵感
    // 构建 Bulk API 请求体（NDJSON 格式）
    // 一次 HTTP 请求提交全部文档
}
```

- 原有的 `sync()` 实时同步仍保留（创建/更新时触发）
- ES 未配置时自动跳过

### 4. ES search_after 游标分页

**问题**：`from + size` 深分页性能随偏移量线性下降

**方案**：改用游标分页，`sort` 增加 `id` 作为唯一 tiebreaker

```json
{"search_after": [13, "198324536624549889"],
 "sort": [{"heat": {"order": "desc"}}, {"id": {"order": "desc"}}]}
```

| 文件 | 改动 |
|------|------|
| `SearchResultVO.java` | 新增 `sortValue` 字段 |
| `EsSearchService.java` | `buildQuery()` 支持 `search_after` |
| `Search.vue` | 游标翻页逻辑 |

## 三、新增（2026-07-03）

### 12. Gzip 压缩（网关层）

网关自动压缩 JSON 响应，传输体积减少约 60%。

```yaml
# inspire-gateway/application.yml
server:
  compression:
    enabled: true
    mime-types: application/json,text/html,text/css,text/javascript
    min-response-size: 1024
```

- 响应大于 1KB 时自动开启 Gzip
- 前端网络面板：Content-Encoding: gzip

### 13. 静态资源缓存

上传的图片增加 1 天浏览器缓存头，二次访问时不请求服务器。

```yaml
# inspire-core/application.yml
spring:
  web:
    resources:
      static-locations: file:/tmp/inspire-uploads/
      cache:
        period: 86400
```

- `86400` 秒 = 1 天
- 缓存期间浏览器直接从本地加载，不触发 HTTP 请求

### 14. 冷热分离：只读事务标记

为读密集型方法添加 `@Transactional(readOnly = true)`，数据库可针对只读事务进行优化（减少锁、允许走快照读等）。

| 方法 | 添加前 | 添加后 |
|------|--------|--------|
| `listPublic()` | 无事务 | `@Transactional(readOnly = true)` |
| `recommend()` | 无事务 | `@Transactional(readOnly = true)` |
| `getDetail()` | 无事务 | `@Transactional(readOnly = true)` |
| `listVersions()` | 无事务 | `@Transactional(readOnly = true)` |
