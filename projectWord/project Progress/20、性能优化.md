# 灵思集 — 性能优化

> 版本: v2.0 | 更新日期: 2026-07-02

---

## 已实施

### 1. HikariCP 连接池调优

调整数据库连接池参数，减少连接争抢：

```yaml
spring:
  datasource:
    hikari:
      maximum-pool-size: 15
      minimum-idle: 5
      connection-timeout: 3000
      max-lifetime: 1800000
```

### 2. 图片缩略图

上传图片时自动生成 400px 宽的缩略图，列表页可使用缩略图代替原图：

```java
// FileController.java - 文件上传后自动生成
ImageIO.write(thumb, "jpg", new File(dir, "thumb_" + filename));
```

- 原图：`/uploads/xxx.jpg`（用于详情页）
- 缩略图：`/uploads/thumb_xxx.jpg`（用于列表页）

上传响应新增 `thumbUrl` 字段。

### 3. Redis 连接配置

添加 Redis 连接配置，为后续缓存做准备：

```yaml
spring:
  data:
    redis:
      host: localhost
      port: 6379
      password: '123456'
inspire:
  redis:
    ttl-seconds: 30
```

### 4. 前端代码拆分（Code Splitting）

`vite.config.js` 将 Element Plus 和 Vue 拆分为独立 chunk：

```js
manualChunks: {
  'element-plus': ['element-plus'],
  'vendor': ['vue', 'vue-router']
}
```

| 指标 | 优化前 | 优化后 |
|------|--------|--------|
| 主 chunk | 1,094 kB | 65 kB |
| 缓存策略 | 单文件，无法缓存 | element-plus / vendor 可独立缓存 |

### 5. 图片懒加载

所有 `<img>` 标签添加 `loading="lazy"`，浏览器原生支持，节省带宽 30-50%。

### 6. 页面过渡动画

页面切换添加淡入 + 位移过渡，时长 150ms，`mode="out-in"`，体感流畅。

## 待实施

### 7. Redis 缓存读接口

待实现：`listPublic()` 和 `recommend()` 方法接入 Redis 缓存，30 秒 TTL。

```java
// 预期实现
String cacheKey = "pub:" + page + ":" + tag + ":" + sort;
String cached = redisTemplate.opsForValue().get(cacheKey);
if (cached != null) return parseResult(cached);
// query DB and cache
redisTemplate.opsForValue().set(cacheKey, jsonResult, 30, TimeUnit.SECONDS);
```

难点：VO 中包含每用户的 `liked`/`collected` 状态，缓存方案需按登录态分离。

### 8. N+1 查询消除

`List<InspireVO> toVO()` 对列表每项单独查询昵称（20 项 = 21 次 SQL）。

```sql
SELECT nickname FROM user WHERE id = ?   -- 每项一次
```

方案：批量查询后用 Map 传递：

```java
Map<Long, String> nicknames = jdbcTemplate.query(
    "SELECT id, nickname FROM user WHERE id IN (?,?,?)", ...);
// 传入 toVO，跳过逐条查询
```

### 9. 数据库索引

hot list / 分类浏览缺少覆盖排序字段的复合索引：

```sql
ALTER TABLE inspire_main ADD INDEX idx_status_deleted_time (status, deleted, create_time DESC);
ALTER TABLE inspire_main ADD INDEX idx_status_deleted_heat (status, deleted, heat DESC);
ALTER TABLE inspire_main ADD INDEX idx_tag_status_heat (tag, status, deleted, heat DESC);
```
