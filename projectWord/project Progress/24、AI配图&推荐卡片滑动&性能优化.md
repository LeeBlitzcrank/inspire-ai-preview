## 一、AI 配图改进

### 问题
原 AI 配图使用 `picsum.photos` 随机占位图服务，关键词和图片完全无关，对用户体验无实际帮助。

### 解决方案
集成 **Unsplash API**，根据关键词搜索真实图片。

#### 后端改动
- `InspireController.suggestImages()` — 新增 Unsplash 搜索逻辑
  - 配置了 `unsplash.access-key` 时调用 Unsplash API 搜索真实图片
  - 未配置 key 或 API 失败时自动回退到 picsum 占位图
- 关键词处理
  - **全角→半角转换**（`：` → `:`，`，` → `,` 等），避免全角标点导致 API 返回 410
  - **空格编码为 `%20`**（URLEncoder.encode 默认将空格转为 `+`，但在 URI 中应使用 `%20`）
  - 关键词截短到 80 字符
- HTTP 客户端强制使用 **HTTP/1.1**（避免 Java HttpClient 默认 HTTP/2 导致的问题）

#### 配置方式
```yaml
inspire:
  unsplash:
    access-key: YOUR_UNSPLASH_ACCESS_KEY
```
在 [unsplash.com/developers](https://unsplash.com/developers) 注册应用获取 Access Key。

## 三、性能优化

### 1. N+1 查询 → 批量查询

**`InspireServiceImpl.toVO()`** 对列表中每一条 `InspireMain` 都执行：
- `SELECT nickname FROM user WHERE id = ?`
- `SELECT ... FROM collect_? WHERE user_id = ? AND inspire_id = ?`
- `SELECT ... FROM inspire_like_? WHERE inspire_id = ? AND user_id = ?`

列表 20 条 + 已登录 → **60 次额外查询**。改为 `toVOList()` 批量查询方法：
- **Nickname**：`WHERE id IN (...)` 一次查完
- **收藏**：`SELECT inspire_id FROM collect_{shard} WHERE user_id = ? AND inspire_id IN (...)`
- **点赞**：按 `inspire_id % 10` 分组，每分片一次 IN 查询（≤10 次）

**效果：60 次 → ≤12 次**

### 2. 用户列表分页
`listMyPublished` / `listMyDrafts` / `listMyCollects` 原为全量 `selectList`，增加 `page`/`size` 参数。

### 3. SQL 拼接 → MyBatis-Plus Page
```java
// 之前（有注入风险）
w.last("LIMIT " + size + " OFFSET " + off);

// 之后（参数化查询）
Page<InspireMain> mpPage = mainMapper.selectPage(new Page<>(page, size), wrapper);
```

### 4. 缓存 Key 补全
`@Cacheable(key = "#query.page + ':' + #query.size + ':' + #query.tag + ':' + #query.sort")` — 补上缺失的 `#query.size`。

### 5. PaginationInnerInterceptor 补全
MyBatis-Plus 原配置只有 `DynamicTableNameInnerInterceptor`，缺少 `PaginationInnerInterceptor`，导致 `selectPage` 的 COUNT 查询不执行，`total` 始终为 0。现已补全。

## 四、文件上传修复

### CORS / SSL 问题
`FileController.uploadFromUrl()` 需要从外部 AI 图片 URL 下载图片，但：
1. `HttpURLConnection` 默认 SSL 握手失败（外部服务证书不受信任）
2. 新增 `getTrustAllSslFactory()` 方法信任所有证书（开发环境下） + 放通 `HostnameVerifier`

### 前端硬编码 URL 提取
- `Create.vue` 中 `http://localhost:8083` 硬编码改为 Vite proxy (`/uploads` → `localhost:8083`)
- 部署到其他环境无需改代码
