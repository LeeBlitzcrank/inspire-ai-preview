商业化的话，思考逻辑要变一下——从"功能完整"转向"用户增长、留存、变现、合规"。

## 商业化优先级

### P0 — 上线前必须做（合规 & 基础体验）

**1. 内容审核（敏感词 + 图片审核）**
法律要求。发布前自动检查敏感词、图片鉴黄/暴恐。可以在 inspire-core 加一个审核过滤器，通不过的自动进入"审核中"状态，管理员后台审核。

**2. Docker Compose + 一键部署**
不能再手动启动 7 个后端 + 前端 + Docker 容器。`docker compose up` 全套起来，否则运维成本太高。

**3. 数据看板**
管理员需要知道：日活用户、灵感发布量、点赞收藏数据、注册转化率。没有数据没法做决策。

**4. 用户协议 + 隐私政策**
法律必备。

### P1 — 用户增长 & 留存

**5. 关注系统 + 关注流**
社区的核心。用户关注创作者后，首页变成关注流。这决定了用户会不会天天打开。

**6. 消息通知**
被点赞、收藏、评论、关注时收到通知。可以用 WebSocket 或轮询，前端加小红点。不做通知，用户流失率很高。

**7. 热门榜单 / 推荐流**
目前首页是分类浏览，缺一个"热门推荐"Feed 流。基于点赞数 + 时间衰减 + Flink 实时算热度。

**8. 分享裂变**
分享到微信/朋友圈时生成精美的卡片海报。这是最便宜的获客方式。

### P2 — 商业化变现

**9. VIP 会员体系**
- 免费用户：每天限 5 次 AI 探索、普通搜索
- VIP 用户：无限 AI、高级搜索、去广告、优先推荐
- 支付接入：微信支付 / 支付宝

**10. 创作者激励**
灵感被点赞多 → 获得积分 → 兑换曝光机会 / 提现。激励优质内容产出。

**11. 平台抽佣 / 广告位**
灵感详情页插 Banner 广告、推荐流插入商单内容。

### P3 — 运营工具

**12. 用户举报 + 处理流程**
用户举报不良内容 → 管理员收到通知 → 审核/下架。不做这个平台容易被滥用。

**13. 内容精选 / 官方推荐**
管理员可以把高质量灵感设为"编辑推荐"，在首页展示。

**14. SEO**
灵感详情页生成静态描述，让百度能搜到。这是免费的流量来源。

# 灵思集 --- 内容审核 & 数据看板功能文档

## 一、内容审核

### 1.1 功能概述

灵感发布时自动进行文本审核，检测敏感词、手机号、外部链接。命中规则时提示用户，不允许发布。

### 1.2 敏感词过滤器

类：TextFilter.java（inspire-core 模块）

检测规则：

| 规则 | 说明 | 示例 |
|------|------|------|
| 敏感词 | 预设敏感词列表 | 赌博、色情、毒品、诈骗、传销 |
| 手机号 | 正则匹配 11 位手机号 | 13800138000 |
| 外部链接 | 正则匹配 http/https URL | https://example.com |

触发后的表现：创建灵感时如果内容命中规则，后端返回 BusinessException，前端弹窗提示违规原因。

### 1.3 接入方式

在 InspireServiceImpl.create() 中调用：

```java
String reason = TextFilter.check(req.getTitle());
if (reason == null) reason = TextFilter.check(req.getContent());
if (reason != null) {
    throw new BusinessException("内容审核未通过: " + reason);
}
```

## 二、数据看板

### 2.1 功能概述

管理后台的监控大屏，展示平台核心运营指标：日活、发布量、互动数据、注册增长。

入口：管理后台（/admin/login）-> 数据看板

### 2.2 指标说明

**核心指标**

| 指标 | 数据来源 | 说明 |
|------|---------|------|
| 日活用户 (DAU) | inspire_main + inspire_comment | 今日有发布/评论的独立用户数 |
| 灵感总数 | inspire_main | 已发布且未删除 |
| 用户总数 | user | 所有未删除用户 |
| AI 调用次数 | ai_call_log | 今日 DeepSeek API 调用次数 |

**今日数据**

| 指标 | 数据来源 | 说明 |
|------|---------|------|
| 今日发布 | inspire_main | DATE(create_time) = CURDATE() |
| 今日注册 | user | DATE(create_time) = CURDATE() |
| 今日点赞 | inspire_like_0~9 | 10 张分表汇总 |
| 今日收藏 | collect_0~9 | 10 张分表汇总 |

**累计数据**

| 指标 | 数据来源 | 说明 |
|------|---------|------|
| 累计点赞 | inspire_like_0~9 | 所有分表 COUNT 总和 |
| 累计收藏 | collect_0~9 | 所有分表 COUNT 总和 |

**分类 & 城市**

| 面板 | 数据来源 | 说明 |
|------|---------|------|
| 分类热度 | inspire_main GROUP BY tag | 各分类灵感数量 |
| 城市分布 | inspire_main GROUP BY publish_city | 各城市灵感数量 |

### 2.3 后端实现

接口：GET /admin/dashboard

类：

| 类 | 模块 | 说明 |
|------|------|------|
| DashboardVO | inspire-admin/dto | 看板返回体 |
| AdminDashboardServiceImpl | inspire-admin/service | 看板数据查询 |
| AdminDashboardService | inspire-admin/service | 接口 |

### 2.4 AI 调用统计

表：ai_call_log

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 雪花ID |
| call_date | DATE | 调用日期 |
| keyword | VARCHAR | 用户关键词 |
| user_id | BIGINT | 用户ID |
| create_time | DATETIME | 调用时间 |

记录流程：
1. AiServiceImpl 成功调 DeepSeek API 后
2. 异步 POST http://localhost:8083/internal/ai/call
3. InternalController 写入 ai_call_log
4. 看板查询 SELECT COUNT(*) FROM ai_call_log WHERE call_date = CURDATE()
