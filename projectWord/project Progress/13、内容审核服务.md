商业化的话，思考逻辑要变一下——从"功能完整"转向"用户增长、留存、变现、合规"。

## 商业化优先级

### P0 — 上线前必须做（合规 & 基础体验）

**1. 内容审核（敏感词 + 图片审核）**
法律要求。发布前自动检查敏感词、图片鉴黄/暴恐。可以在 inspire-core 加一个审核过滤器，通不过的自动进入"审核中"状态，管理员后台审核。

**2. Docker Compose + 一键部署**
不能再手动启动 7 个后端 + 前端 + Docker 容器。`docker compose up` 全套起来，否则运维成本太高。

**3. 数据看板**
管理员需要知道：日活用户、灵感发布量、点赞收藏数据、注册转化率。没有数据没法做决策。

**4. 用户协议 + 隐私政策**
法律必备。

### P1 — 用户增长 & 留存

**5. 关注系统 + 关注流**
社区的核心。用户关注创作者后，首页变成关注流。这决定了用户会不会天天打开。

**6. 消息通知**
被点赞、收藏、评论、关注时收到通知。可以用 WebSocket 或轮询，前端加小红点。不做通知，用户流失率很高。

**7. 热门榜单 / 推荐流**
目前首页是分类浏览，缺一个"热门推荐"Feed 流。基于点赞数 + 时间衰减 + Flink 实时算热度。

**8. 分享裂变**
分享到微信/朋友圈时生成精美的卡片海报。这是最便宜的获客方式。

### P2 — 商业化变现

**9. VIP 会员体系**
- 免费用户：每天限 5 次 AI 探索、普通搜索
- VIP 用户：无限 AI、高级搜索、去广告、优先推荐
- 支付接入：微信支付 / 支付宝

**10. 创作者激励**
灵感被点赞多 → 获得积分 → 兑换曝光机会 / 提现。激励优质内容产出。

**11. 平台抽佣 / 广告位**
灵感详情页插 Banner 广告、推荐流插入商单内容。

### P3 — 运营工具

**12. 用户举报 + 处理流程**
用户举报不良内容 → 管理员收到通知 → 审核/下架。不做这个平台容易被滥用。

**13. 内容精选 / 官方推荐**
管理员可以把高质量灵感设为"编辑推荐"，在首页展示。

**14. SEO**
灵感详情页生成静态描述，让百度能搜到。这是免费的流量来源。
