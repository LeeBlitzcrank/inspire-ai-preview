## 一、多图 / 灵感图册

灵感支持上传多张图片，详情页以轮播形式展示。

### 数据库

```sql
ALTER TABLE `inspire_main` ADD COLUMN `images` TEXT COMMENT '多图JSON数组' AFTER `img`;
```

`images` 字段存储 JSON 数组，如 `["url1.jpg","url2.jpg"]`。

### 后端

- `InspireMain.java` / `InspireVO.java` / `InspireCreateRequest.java` / `InspireUpdateRequest.java` 均新增 `images` 字段
- `InspireServiceImpl.toVO()` 使用 `ObjectMapper` 解析 JSON 数组为 `List<String>`
- 创建/更新时透传 `images` 原始 JSON 字符串

### 前端

- **Create.vue**：上传区改为多图网格（3列自适应），每张带删除按钮，「+ 添加图片」可继续上传
- **InspireDetail.vue**：单图保持原样，多图切换为轮播（左右箭头 + 圆点指示器）
- 响应式布局：`grid-template-columns: repeat(auto-fill, minmax(100px, 1fr))`

## 三、版本历史

用户编辑已发布灵感时，自动保存旧版本，供其他用户查阅。

### 数据库

```sql
CREATE TABLE `inspire_version` (
  `id` BIGINT NOT NULL COMMENT '雪花ID',
  `inspire_id` BIGINT NOT NULL COMMENT '关联灵感ID',
  `version_number` INT NOT NULL COMMENT '版本号',
  `title` VARCHAR(120) NOT NULL,
  `content` TEXT,
  `img` VARCHAR(255) DEFAULT '',
  `images` TEXT,
  `tag` VARCHAR(30) DEFAULT '',
  `change_summary` VARCHAR(200) DEFAULT '',
  `create_time` DATETIME DEFAULT CURRENT_TIMESTAMP,
  PRIMARY KEY (`id`),
  KEY `idx_inspire_id` (`inspire_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### 后端 API

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/inspire/{id}/versions` | 版本历史列表 |
| GET | `/inspire/{id}/versions/{versionId}` | 指定版本详情 |

### 触发时机

编辑已发布灵感 → 点击「📤 发布」→ 自动保存旧版本到 `inspire_version` 表

### 前端

- 灵感详情页评论区上方显示 📜 **版本历史** 区域
- 点击版本项 → 弹出预览弹窗 → 显示版本号、标题、时间、正文内容
- 点击遮罩或 ✕ 关闭

## 四、编辑模式 & 草稿管理

### 编辑已发布灵感

- 灵感详情页显示 ✏️ **编辑** 按钮（仅作者可见）
- 点击进入 `/edit/:id` 路由，复用 Create.vue 组件
- 表单自动预填标题、分类、正文、图片
- 两个按钮：
  - **📤 发布** — 更新灵感并保存旧版本
  - **💾 保存到草稿** — 改为草稿状态（status=0）

### 草稿管理

- 个人中心 → **我的草稿** Tab，调用 `GET /inspire/my/drafts`
- 支持编辑草稿 → 发布
- `status=0` 的灵感不出现于公共列表和搜索结果

### 前端路由

| 路径 | 组件 | 说明 |
|------|------|------|
| `/create` | Create.vue | 创建新灵感 |
| `/edit/:id` | Create.vue | 编辑已有灵感（editId 模式） |

### AI 探索区

编辑模式下自动隐藏 AI 探索区（`v-if="!editId"`）

## 六、用户 ID BigInt 精度修复

### 问题

登录时 `TokenResponse.userId` 以 JSON 数字形式返回（如 `198275646181347329`）。Snowflake ID 超过 JavaScript 安全整数上限（2^53 ≈ 9e15），导致四舍五入成 `198275646181347330`。

### 修复

- `Login.vue`：不再从 API 响应取 `userId`，改为解码 JWT 的 `sub` 字段
- `InspireDetail.vue`：`currentUserId` computed 改为从 JWT 解码

---

## 七、时间显示修复

### 问题

MySQL Docker 容器运行在 UTC 时区，JDBC 驱动在读取 `DATETIME` 时偏移 8 小时。

### 修复

1. JDBC URL 添加 `connectionTimeZone=%2B08:00`
2. Java 代码显式设置 `createTime`：
   ```java
   LocalDateTime.now(java.time.ZoneId.of("Asia/Shanghai"))
   ```
3. 前端 `formatTime` 处理数组格式 + 时间戳格式
