# 灵思集 --- 评论系统 & 用户中心功能文档

## 一、评论系统

### 1.1 功能概述

灵感详情页底部提供完整的评论区，支持发表评论、回复、删除评论和评论头像显示。

**入口**：任意灵感详情页（/detail/:id）→ 底部评论区

### 1.2 数据模型

表结构：inspire_comment

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT (Snowflake) | 评论ID，序列化为字符串 |
| inspire_id | BIGINT | 所属灵感ID |
| user_id | BIGINT | 评论人ID，序列化为字符串 |
| username | VARCHAR(60) | 评论时用户名 |
| avatar | VARCHAR(255) | 评论时头像 |
| content | VARCHAR(500) | 评论内容 |
| parent_id | BIGINT | 0=顶级评论，>0=回复目标评论ID |
| reply_user_id | BIGINT | 被回复用户ID |
| reply_username | VARCHAR(60) | 被回复用户名 |
| create_time | DATETIME | 创建时间 |
| deleted | TINYINT | 逻辑删除 |

---

### 1.3 后端接口

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /inspire/{id}/comments?page=1&size=20 | 分页查询评论列表 |
| POST | /inspire/{id}/comment | 发表评论（带回复信息） |
| DELETE | /inspire/{id}/comment/{commentId} | 删除评论（本人+级联删除子回复） |

请求体（POST /inspire/{id}/comment）：
```json
{
  "content": "评论内容",
  "username": "昵称",
  "avatar": "头像",
  "parentId": 0,
  "replyUserId": 0,
  "replyUsername": ""
}
```

响应体：
```json
{
  "code": 200,
  "data": {
    "records": [{
      "id": "197630734893060097",
      "userId": "197222502466850817",
      "username": "昵称",
      "avatar": "\U0001f60a",
      "content": "评论内容",
      "parentId": "0",
      "replyUserId": "0",
      "replyUsername": "",
      "createTime": "2026-06-30T12:00:00",
      "children": []
    }],
    "total": 1
  }
}
```

> 注意：所有 Long 类型 ID 均通过 @JsonSerialize(using = ToStringSerializer.class) 序列化为字符串，避免 JavaScript 精度丢失。

### 2.3 城市选择器

组件：省、市 两级级联选择器（el-cascader）

数据文件：frontend/src/utils/cityData.js

覆盖范围：全国所有地级市（33个省级单位、约350个城市）

自动定位：
1. 打开编辑弹窗时调用 /api/auth/ip-location
2. 后端从 X-Forwarded-For 头获取客户端IP
3. 调用 ip-api.com 查询IP所在城市
4. 查询失败默认返回北京

英文到中文城市名映射：enToCn 对象，覆盖所有地级市的英文名到中文名。

## 五、重要设计决策

### 5.1 Snowflake ID 精度
所有 ID 序列化为字符串，避免 JavaScript 的 Number 类型无法精确表示 19 位 Snowflake ID。

### 5.2 评论用户信息的反范式存储
评论表直接存储 username 和 avatar，不与用户表实时关联。修改资料后已发表的旧评论不更新。

### 5.3 后台管理员与普通用户隔离
admin_user 表和 user 表独立，请求拦截器根据 URL 路径自动选择使用 adminToken 或 token。

### 5.4 自动定位的本地限制
本地开发时客户端IP为 127.0.0.1，无法获取真实地理位置。部署到有公网IP的服务器后可正常工作。
