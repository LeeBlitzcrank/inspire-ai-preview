
**消息队列要做的事：**
1. Docker 部署 RocketMQ（NameServer + Broker）
2. 新建 `inspire-mq` 公共模块（生产消息 + 消费消息）
3. 用户注册 → 发 `topic_user_register` 消息
4. 灵感发布/收藏/点赞 → 发 `topic_user_behavior` 消息
5. 消费者打印日志（未来替换成 Flink）

第一步：Docker 部署 RocketMQ
# 拉镜像
docker pull apache/rocketmq:5.2.0

# 创建 broker 配置
mkdir -p ~/docker/rocketmq/conf
cat > ~/docker/rocketmq/conf/broker.conf << 'EOF'
namesrvAddr=localhost:9876
brokerIP1=localhost
brokerId=0
autoCreateTopicEnable=true
EOF

# 启动 NameServer（注册中心）
docker run -d --name rmq-namesrv \
-p 9876:9876 \
apache/rocketmq:5.2.0 \
sh mqnamesrv

# 等几秒，启动 Broker（消息服务器）
sleep 3
docker run -d --name rmq-broker \
-p 10911:10911 -p 10909:10909 \
-v ~/docker/rocketmq/conf/broker.conf:/home/rocketmq/rocketmq-5.2.0/conf/broker.conf \
-e "NAMESRV_ADDR=localhost:9876" \
apache/rocketmq:5.2.0 \
sh mqbroker -c /home/rocketmq/rocketmq-5.2.0/conf/broker.conf

# 确认启动
sleep 2
docker logs rmq-namesrv --tail 3
docker logs rmq-broker --tail 3

## MQ 消息队列 — 交付总结

### 模块结构

```
inspire-mq (无端口，公共模块)
├── constant/MqTopicConstants.java   5个 Topic 常量
├── producer/MqProducer.java         统一消息发送器
└── consumer/ (待加)
```

### 消息发送覆盖

| 场景 | Topic | 发送方 |
|------|-------|--------|
| 用户注册 | `topic_user_register` | Auth 服务 |
| 用户登录 | `topic_user_behavior` | Auth 服务 |
| 灵感发布 | `topic_inspire_publish` | Core 服务 |
| 收藏灵感 | `topic_user_behavior` | Core 服务 |
| 点赞灵感 | `topic_user_behavior` | Core 服务 |

### RocketMQ 部署（你先跑起来）

```bash
# 创建 broker 配置
# 停掉旧的
docker rm -f rmq-broker

# 重建 broker.conf
mkdir -p ~/docker/rocketmq/conf
cat > ~/docker/rocketmq/conf/broker.conf << 'EOF'
namesrvAddr=localhost:9876
brokerIP1=localhost
brokerId=0
autoCreateTopicEnable=true
EOF

# 启动 Broker（挂载配置文件）
docker run -d --name rmq-broker \
  -p 10911:10911 -p 10909:10909 \
  -v ~/docker/rocketmq/conf/broker.conf:/home/rocketmq/rocketmq-5.2.0/conf/broker.conf \
  -e "NAMESRV_ADDR=localhost:9876" \
  apache/rocketmq:5.2.0 \
  sh mqbroker -c /home/rocketmq/rocketmq-5.2.0/conf/broker.conf

sleep 2
docker logs rmq-broker --tail 3
```

RocketMQ 跑起来后重启 auth 和 core，`MqProducer` 会自动检测到 RocketMQ 并用反射调用客户端发送消息。

等 `.m2` 缓存锁的问题解决后，再把消费者类加回来就能消费消息了。
