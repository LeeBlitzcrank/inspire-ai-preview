先确认 RocketMQ 跑着，然后等 RocketMQ 跑起来了，重启 auth 和 core。然后做几个操作触发消息，最后用 RocketMQ 的命令行查消息：

**1. 确认 RocketMQ 就绪**
```bash
# 看 NameServer 和 Broker 日志
docker logs rmq-namesrv --tail 3
docker logs rmq-broker --tail 3
```

**2. 重启 auth + core**
```bash
mvn spring-boot:run -pl inspire-auth
mvn spring-boot:run -pl inspire-core
```

启动时如果 RocketMQ 可用，auth/core 日志里会打印：
```
RocketMQ已配置: namesrv=localhost:9876，消息将异步发送
```

**3. 做几个操作**
用之前的一个 curl 脚本，或者直接操作浏览器：

```bash
# 注册（发 topic_user_register）
TOKEN=$(curl -s -X POST http://localhost:8080/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{"username":"mqtest","password":"123456","confirmPassword":"123456","email":"mq@test.com"}' \
  | python3 -c "import sys,json;print(json.load(sys.stdin)['data']['token'])")

# 创建灵感（发 topic_inspire_publish）
curl -s -X POST http://localhost:8080/api/inspire \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"title":"测试MQ消息","content":"测试内容","tag":"美食","status":1}'

# 收藏（发 topic_user_behavior type=collect）
curl -s -X POST http://localhost:8080/api/inspire/197219830972026881/collect \
  -H "Authorization: Bearer $TOKEN"
```

**4. 查 RocketMQ 消息**
```bash
# 查看有哪些 topic
docker exec rmq-broker sh mqadmin topicList -n localhost:9876

# 查看 topic_user_register 的消息（消费者状态）
docker exec rmq-broker sh mqadmin consumerProgress -n localhost:9876 -g consumer_register

# 如果上面命令报错，尝试：
docker exec -it rmq-broker bash
# 容器里执行：
cd /home/rocketmq/rocketmq-5.2.0/bin
sh mqadmin topicList -n localhost:9876
```

如果 MQ 正常工作，`topicList` 里应该能看到 `topic_user_register`、`topic_user_behavior`、`topic_inspire_publish`。

不过由于消费者还没加回来（`.m2` 锁问题），消息会堆积在 RocketMQ 里，等以后加上消费者就能消费了。

更直观的方式：装一个 **RocketMQ Dashboard** 图形界面来看消息：

docker network create rocketmq-net

```bash
docker run -d --name rmq-dashboard \
--network rocketmq-net \
-p 8088:8082 \
-e "JAVA_OPTS=-Drocketmq.namesrv.addr=rmq-namesrv:9876 -Dcom.rocketmq.sendMessageWithVIPChannel=false" \
apacherocketmq/rocketmq-dashboard:latest
```

docker network connect rocketmq-net rmq-namesrv
docker network connect rocketmq-net rmq-broker

# 看日志有没有连接报错
docker logs -f rmq-dashboard

然后浏览器打开 `http://localhost:8088` 就能看到 topic、消息量、消费者状态了。
