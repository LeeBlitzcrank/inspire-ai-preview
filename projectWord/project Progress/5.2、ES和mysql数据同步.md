## 发布自动同步 ES — 已实现

### 改动文件

| 文件 | 改动 |
|------|------|
| `inspire-core/pom.xml` | 新增 `elasticsearch-rest-client` 依赖 |
| `inspire-core/.../es/EsSyncService.java` | **新文件** — ES 同步服务 |
| `inspire-core/.../InspireServiceImpl.java` | `create`/`update`/`deleteById` 后调 `EsSyncService` |
| `inspire-core/application-dev.yml` | 新增 `inspire.es.hosts` 配置，统一数据库名为 `inspire_ai_preview` |

### 同步机制

```
create(灵感) → MySQL写入 → EsSyncService.sync() → ES写入
update(灵感) → MySQL更新 → EsSyncService.sync() → ES更新
delete(灵感) → MySQL软删除 → EsSyncService.delete() → ES删除
```

- 同步是**异步安全的** — ES 失败只打 warn 日志，不影响主流程
- **ES 地址未配置时自动跳过**（`isEnabled()` 检查 `inspire.es.hosts` 是否为空）

### 测试

重启 core 服务后，创建一个新灵感，它就会自动写入 ES：

```bash
# 重启 core 服务
mvn spring-boot:run -pl inspire-core

# 另一个终端测试
TOKEN=$(curl -s -X POST http://localhost:8080/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"alice","password":"123456"}' \
  | python3 -c "import sys,json;print(json.load(sys.stdin)['data']['token'])")

# 创建灵感（自动同步到 ES）
curl -s -X POST http://localhost:8080/api/inspire \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"title":"夏日清爽沙拉","content":"水果蔬菜搭配...","tag":"美食","status":1,"publishCity":"上海"}'

# 搜索（应该能搜到，source: "es"）
curl -s --get "http://localhost:8080/api/search/public" \
  --data-urlencode "keyword=夏日"
```
