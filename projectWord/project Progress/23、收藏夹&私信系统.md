## 一、收藏夹/合集

### 功能概述
用户可以把收藏的灵感归类到文件夹中。所有收藏操作都会弹出文件夹选择器，用户必须选择一个文件夹或新建后才能收藏。

### 重要设计决策
每次收藏必须选择或新建文件夹，不提供直接收藏到"未分类"的路径。如果用户还没有文件夹，系统会自动创建一个名为「默认收藏」的文件夹。

### 数据模型

**collect_folder 表**
| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 主键（字符串序列化） |
| user_id | BIGINT | 所属用户 |
| name | VARCHAR(50) | 文件夹名称 |
| icon | VARCHAR(10) | 图标 emoji |
| sort_order | INT | 排序 |

**collect_x 表（0~9 分表）新增字段**
`folder_id BIGINT` — 关联到的文件夹 ID，NULL 表示未分类

### 后端 API

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | /inspire/collect/folder | 创建收藏夹 |
| GET | /inspire/collect/folders | 列出我的收藏夹 |
| DELETE | /inspire/collect/folder/{id} | 删除收藏夹（同时清空分表中 folder_id） |
| PUT | /inspire/collect/folder/{id} | 重命名收藏夹 |
| POST | /inspire/collect/{id}/folder | 收藏到指定文件夹 |
| PUT | /inspire/collect/{id}/move | 移动收藏到另一个文件夹 |
| GET | /inspire/collect/list?folderId= | 按文件夹获取收藏列表（null=未分类） |

### 前端页面

**Collections.vue** (`/collections`，需登录)
- 顶部：返回按钮 + 标题 + 新建按钮
- 文件夹网格：展示所有文件夹 + 「📂 未分类」卡片
- 选中文件夹后展示该文件夹内的灵感卡片列表
- 每个卡片下方有「📁 移动到文件夹」按钮（可跨文件夹移动）
- 文件夹支持重命名（弹窗）和删除（确认框）
- 卡片支持点击查看详情和取消收藏

**InspireDetail.vue 收藏流程**
- 点击收藏按钮 → `loadCollectFolders()` → 有文件夹？显示选择器 → 无？创建默认文件夹 → 选择器
- 选择器内可勾选已有文件夹，也可输入名称新建文件夹
- 确认后调用 `collectToFolder(inspireId, folderId)`

**index.vue（首页）收藏流程**
- 灵感列表卡片：点击收藏 → 弹出文件夹选择器（与详情页一致）
- 推荐卡片右滑：自动收藏到第一个文件夹（或创建默认文件夹），不弹窗

**Search.vue（搜索页）收藏流程**
- 搜索结果卡片：点击收藏 → 弹出文件夹选择器（与详情页一致）

**Personal.vue**
- tab 栏「我的收藏夹」（原来叫「我的收藏」），旁边有「管理」链接可跳转到 `/collections`
- 点击「管理」进入 Collections.vue 管理文件夹

### 测试方式
1. 首页灵感卡片 → 点击收藏 → 文件夹选择器弹出
2. 灵感详情 → 点击收藏 → 选文件夹 → 确认
3. 推荐卡片右滑 → 自动收藏，Toast 显示文件夹名
4. 进入 `/collections` → 新建文件夹 → 管理
5. 未分类 → 移动到文件夹
6. 删除文件夹 → 收藏自动变为未分类

## 二、私信系统

### 功能概述
用户之间可以发送私信，支持会话列表、聊天视图、未读计数、左滑删除、清空所有会话。

### 入口点
| 位置 | 方式 |
|------|------|
| 首页顶部导航 | 💬 图标 → `/messages?showList=1`（显示会话列表） |
| 灵感详情页 | 关注按钮旁边「💬 私信」→ 直接进入聊天 |
| 首页关注用户列表 | 用户卡片上「💬 私信」→ 直接进入聊天 |
| 其他入口 | 携带 `convId` 参数直接进入聊天视图 |

### 数据模型

**message_conversation 表**
| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 主键（字符串序列化） |
| user1_id | BIGINT | 用户1（ID 较小者，字符串序列化） |
| user2_id | BIGINT | 用户2（ID 较大者，字符串序列化） |
| last_content | VARCHAR(500) | 最后一条消息 |
| last_time | DATETIME | 最后消息时间 |
| unread_user1 | INT | 用户1未读数 |
| unread_user2 | INT | 用户2未读数 |

**message 表**
| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 主键（字符串序列化） |
| conversation_id | BIGINT | 会话 ID（字符串序列化） |
| from_user_id | BIGINT | 发送者（字符串序列化） |
| to_user_id | BIGINT | 接收者（字符串序列化） |
| content | VARCHAR(1000) | 消息内容 |
| create_time | DATETIME | 发送时间 |

### 技术要点
**大数精度处理**：雪花 ID（如 198658775131885570）超过 JS 的 `Number.MAX_SAFE_INTEGER`，需要在后端实体上加 `@JsonSerialize(using = ToStringSerializer.class)`，让 Jackson 序列化为字符串，前端 JS 才能精确读取。

**会话对象字段**：`MessageConversation` 新增 `targetNickname` 和 `targetUsername`（`@TableField(exist = false)`），在 `getConversations()` 中从 `user` 表查询填充。

### 后端 API

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | /message/send | 发送私信 |
| POST | /message/send-by-username | 按用户名发送 |
| POST | /message/start | 创建或获取会话（按 userId） |
| GET | /message/conversations | 会话列表（含 targetNickname） |
| GET | /message/list | 消息列表（分页） |
| POST | /message/read | 标记已读 |
| GET | /message/unread | 未读数 |
| DELETE | /message/conversation/{id} | 删除单个会话 |
| DELETE | /message/conversations | 清空所有会话 |

### 前端页面

**Messages.vue** (`/messages`，需登录)

样式基于 chat.html（紫色主题渐变 #6366f1 → #a855f7）：

```
┌─────────────┬──────────────────────────────┐
│  ← 私信 清空  │  ← 👤 用户昵称              │
│─────────────│                              │
│ 👤 用户A     │  消息气泡（左：白/右：紫）    │
│   最后消息   │                              │
│ 👤 用户B     │                              │
│   最后消息   │                              │
│             │  输入框 ───────────────────── │
│             │  [输入消息...    ] [发送]    │
└─────────────┴──────────────────────────────┘
  340px 侧栏                     flex:1
```

**导航与返回逻辑**
- `?showList=1`（从首页 💬 进入）→ 显示会话列表，聊天中点返回回列表
- `?convId=xxx`（从私信按钮进入）→ 直接打开聊天，返回走 `router.back()`
- 桌面端聊天气泡头部也有返回按钮（不依赖移动端）

**交互功能**
- **轮询新消息**：进入会话后每 3 秒拉取新消息（仅增量更新）
- **左滑删除**：会话列表项支持左滑露出红色「删除」按钮（触屏 + 鼠标均可）
- **清空聊天**：会话列表头部红色「清空聊天」按钮，确认后删除所有会话
- **新建消息**：~~「+ 新消息」按钮已移除~~，通过其他入口发起

### 消息发送流程
```
用户 A 从灵感详情点击「💬 私信」
  → POST /message/start（创建或获取 conversation）
  → 返回 convId
  → 跳转 /messages?convId=xxx
  → Messages.vue 加载会话 + 消息
  → 用户输入消息 → POST /message/send
  → 刷新消息列表
```

### 移动端适配
- 会话列表占满屏幕宽度
- 点击会话后聊天面板从右侧滑入（`left:100% → left:0`，300ms 过渡）
- 聊天头部返回按钮回到列表

### 测试方式
1. 使用两个账号分别登录（A 和 B）
2. 账号 A 进入 B 的灵感详情 → 点击「💬 私信」→ 直接进入聊天
3. 账号 A 发送消息 → 账号 B 刷新 `/messages` → 看到未读
4. 验证会话列表显示昵称、最后消息、未读数
5. 验证聊天气泡方向正确
6. 验证左滑删除会话
7. 验证清空所有会话
8. 验证从首页 💬 进入显示列表，从详情进入直接聊天

## 四、新增功能

### 1. 热门标签云

首页显示当前最热门的灵感标签（按灵感数量降序，取前 20 个），点击标签跳转到该分类的灵感列表。

**后端**
- `GET /inspire/hot-tags` → 返回 `[{tag: "美食", count: 268}, ...]`
- 实现：`SELECT tag, COUNT(*) as count FROM inspire_main WHERE deleted=0 AND status=1 GROUP BY tag ORDER BY count DESC LIMIT 20`

**前端**
- `index.vue` — 标签云区域位于推荐卡片上方
- 热门标签按热度大小显示（12px-16px）
- 点击标签 → 切换到「灵感分类」tab，选中该标签并加载灵感列表
- 调用 `loadHotTags()` 在 `onMounted` 时加载

### 2. Markdown 富文本编辑器

灵感创建和详情页支持 Markdown 语法，简化排版。

**前端**
- `Create.vue` — 文本编辑区上方新增 Markdown 工具栏
  - B（加粗）、I（斜体）、H1/H2（标题）、•（列表）、🔗（链接）、&lt;/&gt;（代码块）
  - 工具栏按钮在光标位置或选中文本前后插入 Markdown 语法
- `InspireDetail.vue` — 内容渲染改为 Markdown 转换
  - 移除原来的纯文本段落分割（`v-for="(p, i) in paragraphs"`）
  - 替换为 `v-html="renderedContent"` 渲染 Markdown → HTML
  - 支持：标题 #/##/###、加粗 **text**、斜体 *text*、链接 [text](url)、列表 - item、代码块 ```code```、行内代码 \`code\`
  - 添加 `md-content` 相关 CSS 样式

**支持语法**
| 语法 | 效果 |
|------|------|
| `# 标题` | H1 大标题 |
| `**加粗**` | **加粗** |
| `*斜体*` | *斜体* |
| `[文字](url)` | [链接](url) |
| `- 列表项` | 无序列表 |
| ` ```code``` ` | 代码块 |
| `` `code` `` | 行内代码 |

### 3. AI 自动配图

创建灵感时，点击「🤖 AI 配图」按钮，系统根据标题/内容自动生成图片建议，用户可选择使用。

**后端**
- `POST /inspire/suggest-images` → 根据关键词返回图片 URL 列表
- 当前使用 picsum.photos 生成主题相关占位图（基于关键词哈希种子）

**前端**
- `Create.vue` — Markdown 工具栏右侧新增「🤖 AI 配图」按钮
- 点击后调用 `/api/inspire/suggest-images` 获取 6 张建议图片
- 弹出选择器对话框，可点击预览选择
- 选择后点击「使用此图」→ 下载图片 → 上传到服务器 → 设置为灵感配图
