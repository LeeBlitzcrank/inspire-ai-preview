# 灵思集 — 分享裂变功能

> 版本: v1.0 | 更新日期: 2026-07-02

---

## 一、功能概述

用户可以在灵感详情页分享灵感，记录分享行为并计数，支持多渠道分享裂变。

### 支持的分享方式

| 方式 | 说明 | 适用场景 |
|------|------|----------|
| 复制链接 | 将灵感链接复制到剪贴板 | 桌面端 |
| 分享到其他应用 | 调用系统原生分享（Web Share API） | 移动端 Safari/Chrome |

### 分享计数

- 每次分享行为被记录，`inspire_main.share_count` +1
- 分享按钮实时显示分享次数

---

## 二、数据库

```sql
ALTER TABLE `inspire_main` ADD COLUMN `share_count` INT DEFAULT 0 COMMENT '分享次数' AFTER `collect_count`;
```

---

## 三、后端

### 实体 & VO

- `InspireMain.java` — 新增 `shareCount` 字段
- `InspireVO.java` — 新增 `shareCount` 字段，前端可直接读取

### API

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/inspire/{id}/share` | 记录分享行为，`share_count+1`，并发送 MQ 消息 |

### 实现逻辑

`InspireServiceImpl.share()`：
1. `mainMapper.update` 执行 `SET share_count = COALESCE(share_count, 0) + 1`
2. `mqProducer.send` 发送 `type=share` 的行为消息到 MQ，用于后续数据分析

---

## 四、前端

### 灵感详情页 — 分享按钮

- 显示 `🔗 N` 实时分享次数
- 点击弹出底部分享面板

### 分享面板（底部弹窗）

- 📱 分享到其他应用 — 调用 `navigator.share()`（Web Share API）
- 🔗 复制链接 — 使用 `navigator.clipboard.writeText()`，降级到 `execCommand('copy')`
- 弹窗显示「已分享 N 次」
- 点击遮罩或「取消」关闭

### 数据流

```
用户点击分享 → 调用 POST /inspire/{id}/share → share_count +1
                                      ↓
                             前端 +1 计数器
                                      ↓
                             显示新分享数
```

---

## 五、部署

1. 执行 `ALTER TABLE` 添加 `share_count` 列
2. 重启 `inspire-core`
3. 前端热更新
