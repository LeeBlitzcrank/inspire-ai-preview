# 灵思集 --- 灵感创作功能文档

## 一、功能概述

灵感创作是灵思集的核心功能，允许用户通过 **AI辅助探索** 和 **手动填写** 两种方式生成、编辑并发布灵感内容。

**核心流程：**

```
用户输入关键词 -> AI多级探索 -> 选中内容 -> 补充编辑 -> 发布
                       或
           手动填写 -> 上传封面 -> 发布
```

灵感支持 **语音输入** 转文字，**图片上传** 作为封面，以及 **分类标签** 等元信息。

## 三、AI 多级探索

### 3.1 交互流程

```
用户输入关键词
  -> POST /api/ai/explore {keyword, path}
  -> 返回层级选项 (options[] + summary)
  -> 用户点击一个选项卡片
  -> POST /api/ai/explore (path=上一级路径)
  -> 返回子选项 或 最终内容 (content)
  -> 自动填入表单
```

### 3.2 关键交互

- **探索触发**: 输入关键词, 点击「AI探索」按钮或按 Enter
- **路径选择**: 卡片式排列(2列网格), 每层 3~5 个方向
- **路径拼接**: 用逗号拼接, 例如 `cooking,roast,chicken`
- **层级规则**: 深度<=1 返回子选项, 深度>=2 返回最终内容
- **换一批**: 点击「换一批」携带 refresh=true 跳过 Redis 缓存
- **面包屑**: 点击任意层级可回退重新选择

### 3.3 后端数据流

```
Create.vue -> Gateway -> AiController -> AiServiceImpl
  -> 检查 Redis 缓存 (key=ai:explore:{key}|{path})
  -> 未命中 -> 调用 DeepSeek API
  -> 写入 Redis (TTL=3600s)
  -> 返回 { summary, options[] | content{title,text,tag} }
```

## 四、语音识别

### 4.1 功能说明

灵感详情输入框下方提供语音输入按钮, 用户通过麦克风说话, 系统自动将语音转为文字并追加到灵感详情文本域中。

### 4.2 技术方案

使用浏览器内置的 **Web Speech API** (SpeechRecognition / webkitSpeechRecognition):

```
const SR = window.SpeechRecognition || window.webkitSpeechRecognition
const recognition = new SR()
recognition.lang = 'zh-CN'
recognition.interimResults = true  // 实时中间结果
recognition.continuous = true      // 连续识别
```

### 4.3 交互流程

```
[mic] 点击开始 -> 请求麦克风权限 -> 说话识别 -> 自动追加到详情
[rec] 点击停止 -> 识别结束, 文字已填入
```

### 4.4 浏览器兼容性

| 浏览器  | 支持 | 引擎              |
|---------|------|-------------------|
| Chrome  | 支持 | webkitSpeechRecognition |
| Safari  | 支持 | webkitSpeechRecognition |
| Edge    | 支持 | webkitSpeechRecognition |
| Firefox | 不支持 | Web Speech API 未实现 |

首次使用时会弹出浏览器麦克风权限弹窗, 需点击允许。

### 4.5 错误处理

- 浏览器不支持 -> 提示使用 Chrome/Safari
- 麦克风权限被拒 -> 提示检查麦克风权限
- 识别失败 -> 显示错误信息, 自动停止
- 离开页面 -> 自动释放识别资源

### 4.6 代码位置

`frontend/src/pages/Create.vue` 中的语音识别相关代码:

```
语音识别状态: voiceActive, voiceError
初始化函数: initRecognition()
切换函数: toggleVoice()
生命周期: onUnmounted -> 释放识别资源
```

## 五、手动表单录入

### 5.1 字段说明

| 字段     | 必填 | 类型     | 说明                     |
|----------|------|----------|--------------------------|
| 灵感标题 | 是   | String   | 简短标题                 |
| 所属分类 | 是   | Enum     | 美食/运动/电影/穿搭/文案/旅游/摄影/其他 |
| 灵感详情 | 是   | Text     | 多行文本                 |
| 封面图   | 否   | Image URL| 上传后自动预览           |

### 5.2 表单校验

```javascript
if (!form.value.title)  return ElMessage.warning('请填写标题')
if (!form.value.tag)    return ElMessage.warning('请选择分类')
if (!form.value.content) return ElMessage.warning('请填写灵感详情')
```

## 六、图片上传

### 6.1 上传流程

```
用户点击上传区(虚线框)
  -> 弹出文件选择器 (accept=image/*)
  -> 选择图片 -> POST /api/file/upload (multipart/form-data)
  -> 返回图片URL -> 自动拼接 http://localhost:8083 + url
  -> 预览显示, 支持删除重新上传
```

### 6.2 接口详情

- 请求: `POST /api/file/upload`
- Content-Type: `multipart/form-data`
- 响应: `{ code: 200, data: { url: "/uploads/xxx.jpg" } }`
- 完整访问: `http://localhost:8083/uploads/xxx.jpg`

## 七、后端接口

### 7.1 灵感创建

| 项目     | 说明                                    |
|----------|-----------------------------------------|
| 请求方式 | POST /inspire                          |
| 请求头   | X-Inspire-UserId (用户ID)              |
| 请求体   | { title, tag, content, img, status }   |
| 核心逻辑 | 雪花ID -> MySQL -> ES同步 -> MQ消息    |
| 响应     | { code: 200, data: { id, title, ... } } |

### 7.2 AI 探索

| 项目     | 说明                                    |
|----------|-----------------------------------------|
| 请求方式 | POST /ai/explore                       |
| 请求体   | { keyword, path, refresh }             |
| 响应     | { cacheKey, summary, options[], content{} } |

options 格式:
```json
[
  {"id": "cooking", "label": "家常做法"},
  {"id": "baking",  "label": "烘焙技巧"}
]
```

content 格式(最终层):
```json
{
  "title": "红烧鸡腿的做法",
  "text": "详细步骤...",
  "tag": "美食"
}
```

### 7.3 文件上传

| 项目     | 说明                     |
|----------|--------------------------|
| 请求方式 | POST /file/upload        |
| 请求体   | multipart/form-data      |
| 响应     | { code: 200, data: { url } } |

## 八、完整数据流

```
浏览器 (Create.vue + 语音识别)
    |
    +-- Gateway (8080) -> inspire-ai (8082) -> DeepSeek API + Redis 缓存
    |
    +-- Gateway (8080) -> inspire-core (8083) -> MySQL + ES + RocketMQ
    |                                                  |
    |                                            Flink -> Redis (实时)
    |
    +-- Gateway (8080) -> inspire-search (8086) -> ES
```

## 十、启动步骤

1. 启动依赖:
```bash
docker start my-redis
# 确保 RocketMQ Namesrv + Broker 已启动
```

2. 启动后端:
```bash
cd backend
mvn spring-boot:run -pl inspire-gateway
mvn spring-boot:run -pl inspire-ai
mvn spring-boot:run -pl inspire-core
```

3. 启动前端:
```bash
cd frontend
npm run dev
```

4. 访问: http://localhost:5173/create
