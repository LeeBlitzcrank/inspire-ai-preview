# 灵思集 --- 首页推荐卡片（Tinder 式滑动）

---

## 一、功能概述

首页新增「推荐卡片」模式，以 Tinder 式左右滑动的交互方式浏览灵感推荐。

| 交互 | 动作 | 说明 |
|------|------|------|
| 右滑（拖拽） | 收藏 | 调用收藏接口，卡片向右飞出 |
| 左滑（拖拽） | 跳过 | 卡片向左飞出，切换到下一张 |
| 点击 ♥ 按钮 | 收藏 | 等价于右滑 |
| 点击 ✕ 按钮 | 跳过 | 等价于左滑 |
| 点击卡片文字 | 查看详情 | 跳转到灵感详情页 |

---

## 二、入口变更

### Tab 栏调整
推荐卡片 tab 移至第一位置，页面默认打开即显示推荐卡片。

旧的「💡 为你推荐」tab（仅登录可见）移除，新的「👇 推荐卡片」对所有用户可见（包括未登录）。

Tab 顺序：推荐卡片 → 灵感分类 → 热门排行 → 关注（仅登录）

### 标题文字移除
「全部灵感分类」、「热门排行」、「为你推荐」、「我关注的人」等标题文字全部移除，首页保持简洁。

---

## 三、数据来源

卡片数据从推荐接口 `GET /api/inspire/recommend/list` 获取，取前 20 条。每张卡片展示：

| 字段 | 来源 | 展示方式 |
|------|------|----------|
| 标题 | `i.title` | 卡片中央文字 |
| 分类 | `i.tag` | 卡片背景色（按分类着色） |
| 灵感 ID | `i.id` | 用于收藏、跳转详情 |

### 分类 → 卡片颜色映射

| 分类 | CSS 类 | 背景色 |
|------|--------|--------|
| 美食/饮食 | `.food` | `#fff1e9` |
| 运动/健身 | `.sport` | `#f0fff4` |
| 电影 | `.movie` | `#e8f4ff` |
| 穿搭 | `.wear` | `#fff0f6` |
| 文案/其他 | `.text` | `#f9f0ff` |

## 四、UI 交互

### 卡片布局
- 卡片最大宽度 360px，固定高度 420px
- 圆角 20px，带阴影
- 文字居中展示，支持点击跳转详情

### 拖拽效果
- 拖拽时卡片跟随手指/鼠标偏移
- 偏移超过 100px 触发收藏/跳过动作
- 偏移不到 100px 回弹到原位
- 滑出后 300ms 动画过渡到下一张

### 状态标签
- 右滑时「收藏」标签（绿色）出现在卡片上方，透明度随偏移量渐显
- 左滑时「跳过」标签（红色）出现在卡片上方，透明度随偏移量渐显

## 五、技术实现

### 前端文件
- **页面**: `/frontend/src/pages/index.vue`
- **API**: `getRecommendList` (已封装在 `@/api/inspire`)

### 核心变量

| 变量 | 类型 | 作用 |
|------|------|------|
| `swipeCards` | `ref([])` | 所有卡片数据 |
| `currentIndex` | `ref(0)` | 当前卡片索引 |
| `currentCard` | `computed` | 当前卡片对象 |
| `likeOpacity` | `ref(0)` | 收藏标签透明度 |
| `passOpacity` | `ref(0)` | 跳过标签透明度 |
| `isDragging` | `ref(false)` | 是否正在拖拽 |
| `offsetX` | `ref(0)` | 拖拽偏移量 |

### 关键函数

| 函数 | 触发时机 | 行为 |
|------|----------|------|
| `loadSwipeCards()` | `onMounted` | 从推荐 API 加载卡片数据 |
| `onSwipeStart` | `@mousedown/@touchstart` | 记录起始坐标 |
| `onSwipeMove` | `@mousemove/@touchmove` | 更新卡片偏移和标签透明度 |
| `onSwipeEnd` | `@mouseup/@touchend` | 右滑→收藏，左滑→跳过，不达标→回弹 |
| `swipeLeft` | ✕ 按钮点击 | 跳过当前卡片 |
| `swipeRight` | ♥ 按钮点击 | 收藏当前卡片 |
| `goToCardDetail` | 卡片文字点击 | 跳转灵感详情页 |
| `nextCard` | 滑出动画结束 | 索引+1，循环到第一张 |

### 收藏流程
```
右滑（拖拽 > 100px 或点击 ♥）
  → collectInspire(inspireId)
    → 成功：ElMessage.success('收藏成功')
    → 失败（网络异常）：ElMessage.error('网络异常，收藏失败')
  → 卡片飞出动画（300ms）
  → 切换到下一张
```

## 六、CSS 样式

卡片颜色样式放在全局 `<style>` 块（非 scoped），以支持动态 class 绑定：

```css
.food{background:#fff1e9;color:#e64340;border:1px solid #ffccc7;}
.sport{background:#f0fff4;color:#00b42a;border:1px solid #b7eb8f;}
.movie{background:#e8f4ff;color:#1677ff;border:1px solid #91caff;}
.wear{background:#fff0f6;color:#eb2f96;border:1px solid #ffadd2;}
.text{background:#f9f0ff;color:#722ed1;border:1px solid #d3adf7;}
.btn-left:hover{transform:scale(1.1);}
.btn-right:hover{transform:scale(1.1);}
```

## 八、测试要点

1. 未登录时，拖拽右滑显示「收藏成功」需确认后端收藏接口正常
2. 拖拽回弹效果是否自然
3. 卡片循环到末尾后是否从头开始
4. ♥ 按钮和 ✕ 按钮功能是否与拖拽一致
5. 点击卡片文字是否能正确跳转灵感详情
6. 推荐数据为空时的展示状态
