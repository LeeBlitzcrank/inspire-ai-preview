# 灵思集 — 关注系统 & 关注流功能文档

> 版本: v1.0 | 更新日期: 2026-07-02

---

## 一、功能概述

用户之间可以互相关注。关注后：

- 在首页 **关注 Tab** 中看到被关注用户发布的灵感列表
- 在灵感详情页看到发布者的 **关注/取消关注** 按钮
- 个人中心查看 **我的关注 / 我的粉丝**

---

## 二、数据库

### 表：`user_follow`

| 字段 | 类型 | 说明 |
|------|------|------|
| id | bigint | 主键（雪花算法） |
| follower_id | bigint | 关注者用户 ID |
| followee_id | bigint | 被关注用户 ID |
| create_time | datetime | 关注时间 |
| deleted | tinyint | 逻辑删除 |

- 唯一约束：`(follower_id, followee_id)`

### 表：`user`（已有）

- 前端显示发布者昵称时使用 `user.username` 字段
- 注册时随机生成 4 字昵称（中文）

---

## 三、后端 API

| 方法 | 路径 | 说明 | 认证 |
|------|------|------|------|
| POST | `/inspire/follow/{userId}` | 关注用户 | 需要 |
| DELETE | `/inspire/follow/{userId}` | 取消关注 | 需要 |
| GET | `/inspire/follow/following` | 我的关注列表 | 需要 |
| GET | `/inspire/follow/followers` | 我的粉丝列表 | 需要 |
| GET | `/inspire/follow/feed?followeeId=&page=&size=` | 关注流灵感（分页） | 需要 |
| GET | `/inspire/follow/status/{userId}` | 查询是否已关注 | 需要 |

### 关注流返回结构

```json
{
  "code": 200,
  "msg": "操作成功",
  "data": {
    "records": [ { "id": ..., "title": "...", "username": "...", ... } ],
    "total": 10,
    "current": 1,
    "size": 10
  }
}
```

### 逻辑要点

- 不能关注自己（后端校验，返回业务异常）
- 已关注再次关注 = 业务异常（不能重复关注）
- 未关注去取消 = 业务异常（不能重复取消）
- 所有 ID 使用 `@JsonSerialize(using = ToStringSerializer.class)` 保证前端不丢失精度
- Snowflake ID 长度超过 JS Number 安全范围（> 2^53），必须序列化为字符串

## 四、前端实现

### 4.1 灵感详情页 — 关注按钮

位于发布者信息区，紧挨用户名右侧：

- **未关注**：显示「+ 关注」按钮（蓝色实心）
- **已关注**：显示「已关注」按钮（灰色，hover 变「取消关注」红色）
- 页面加载时调用 `/inspire/follow/status/{userId}` 获取当前状态
- 点击后调用 POST/DELETE 并切换 UI

### 4.2 首页 — 关注 Tab

首页分类 Tab 增加「关注」项（仅在已登录时显示）：

- 点击 -> 显示关注的用户列表（头像 + 用户名）
- 再点击某个用户 -> 显示该用户发布的灵感（调用 `/inspire/follow/feed`）
- 可返回用户列表重选
- 未关注任何人时显示空状态提示

### 4.3 个人中心 — 关注/粉丝

- 用户资料页增加「关注数 / 粉丝数」
- 可点击进入列表页

## 五、注意事项

### 5.1 Snowflake ID 精度

所有返回给前端的 ID 字段必须加 `@JsonSerialize(using = ToStringSerializer.class)`，否则 JS 解析时会丢失精度导致数据错乱。

### 5.2 JWT Token 传递

关注相关 API **不在白名单**（需要认证），前端必须通过 Header 传递 token：
```
Authorization: Bearer <token>
```

### 5.3 重新登录

如果有旧的 JWT token 且密钥发生过变更，需要让用户重新登录。清理 localStorage 中的 token 后登录即可。

### 5.4 清除数据

如果测试数据混乱（比如删表重建导致 ID 不连续），建议清空 `inspire`、`user`、`user_follow` 三张表后重新插入。

### 5.5 已关注状态的渲染

灵感详情页加载时，需调用 `/inspire/follow/status/{publisherId}` 来判断当前用户是否已关注发布者。注意需要避免「自己看自己」的场景——自己发布的灵感不应显示关注按钮。

## 六、测试流程

1. 用两个不同账号分别登录（如 user001、user002）
2. user001 进入 user002 发布的灵感详情页
3. 点击「+ 关注」，按钮变为「已关注」
4. 回到首页，切换到「关注」Tab
5. 点击 user002 的头像，看到 user002 发布的灵感列表
6. 点击灵感进入详情页，确认页面正常
7. 回到灵感详情页，hover「已关注」-> 点击「取消关注」
8. 关注 Tab 中 user002 消失
9. 在个人中心查看关注数 / 粉丝数的变化
